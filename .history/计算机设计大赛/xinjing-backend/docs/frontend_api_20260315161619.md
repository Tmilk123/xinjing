# Frontend API 接口文档

> 更新时间：2026-03-15  
> 服务前缀：`http://127.0.0.1:8000/api/v1`

## 1. 通用约定

- 常见状态码：
  - `200` 请求成功
  - `201` 创建成功
  - `204` 删除成功（无响应体）
  - `400` 参数错误
  - `401` 未认证或 token 无效
  - `403` 已认证但无权限访问
  - `404` 资源不存在
  - `409` 业务冲突（如重复提交）

## 2. 鉴权要求总览

- 免登录：`auth`、`users`、`evaluations`、`chat`、`health`
- 需登录（Bearer Token）：`reports`、`mood-calendar`

说明：

- `reports` 与 `mood-calendar` 已按当前登录用户做数据隔离，不能跨用户读取。
- `users`、`evaluations`、`chat` 当前仍可按 `user_id` 访问，不强制 token。

## 3. Auth

### 3.1 注册

- `POST /auth/register`
- 请求体：

```json
{
  "username": "test001",
  "email": "test001@example.com",
  "phone": "13800138000",
  "password": "12345678",
  "confirm_password": "12345678",
  "nickname": "测试用户",
  "gender": "other",
  "age_range": "25-34"
}
```

- 响应（201）：

```json
{
  "id": 1,
  "username": "test001",
  "email": "test001@example.com",
  "phone": "13800138000",
  "role": "user",
  "status": "active",
  "last_login_at": null
}
```

### 3.2 登录

- `POST /auth/login`
- 请求体：

```json
{
  "username": "test001",
  "password": "12345678"
}
```

- 响应（200）：

```json
{
  "access_token": "<JWT>",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "test001",
    "email": "test001@example.com",
    "phone": "13800138000",
    "role": "user",
    "status": "active",
    "last_login_at": "2026-03-15T08:00:00"
  }
}
```

## 5. Evaluations（辅助筛查）

### 5.1 创建评估会话

- `POST /evaluations/sessions`
- 请求体：

```json
{
  "user_id": 1,
  "screening_type": "phq9",
  "used_modalities": ["face", "voice", "scale", "text"],
  "missing_modalities": []
}
```

- 响应（201）：

```json
{
  "session_id": 10,
  "session_no": "ES-6A173D70AFC7",
  "status": "created",
  "screening_type": "phq9",
  "start_time": "2026-03-15T07:40:18"
}
```

### 5.2 提交问卷

- `POST /evaluations/sessions/{session_id}/submit`
- 请求体：

```json
{
  "template_code": "phq9",
  "answers": [
    { "question_no": 1, "answer_value": 2 },
    { "question_no": 2, "answer_value": 2 },
    { "question_no": 3, "answer_value": 3 },
    { "question_no": 4, "answer_value": 2 },
    { "question_no": 5, "answer_value": 1 },
    { "question_no": 6, "answer_value": 2 },
    { "question_no": 7, "answer_value": 1 },
    { "question_no": 8, "answer_value": 1 },
    { "question_no": 9, "answer_value": 0 }
  ],
  "context": {
    "recent_stress_level": "medium",
    "sleep_status": "normal",
    "appetite_status": "normal",
    "self_evaluation": "stable",
    "social_avoidance_level": "low",
    "remark": "frontend submit"
  },
  "confidence_score": 0.91
}
```

- 响应（200）：

```json
{
  "report_id": 10,
  "session_id": 10,
  "total_score": 14,
  "risk_level": "moderate",
  "confidence_score": 0.91
}
```

### 5.3 查询会话

- `GET /evaluations/sessions?user_id=1&limit=50`
- `GET /evaluations/sessions/{session_id}`

## 6. Reports（需登录）

### 6.1 报告列表

- `GET /reports?limit=50`
- 返回当前登录用户的报告列表

### 6.2 按会话获取报告

- `GET /reports/by-session/{session_id}`

### 6.3 报告详情

- `GET /reports/{report_id}`

### 6.4 前端展示用报告 JSON

- `GET /reports/{report_id}/frontend`
- 响应示例：

```json
{
  "type": "phq9",
  "scale": "PHQ-9",
  "total": 14,
  "max": 27,
  "level": "moderate",
  "color": "#f97316",
  "desc": "存在中度抑郁症状，建议寻求专业心理咨询。",
  "answers": [2, 2, 3, 2, 1, 2, 1, 1, 0],
  "date": "2026-03-15 15:40:20",
  "confidence_score": 0.91
}
```

### 6.5 获取建议

- `GET /reports/session/{session_id}/recommendations`

### 6.6 获取预警

- `GET /reports/alerts?session_id=10`
- 返回当前登录用户预警；可选按 `session_id` 过滤

## 7. Mood Calendar（需登录）

### 7.1 获取情绪日历列表

- `GET /mood-calendar`
- `GET /mood-calendar?month=2026-03`
- 默认返回当前登录用户数据

### 7.2 获取单天记录

- `GET /mood-calendar/{record_date}`
- 示例：`GET /mood-calendar/2026-03-15`

### 7.3 新增或更新单天记录

- `PUT /mood-calendar/{record_date}`
- 请求体：

```json
{
  "mood_key": "sunny",
  "diary_text": "今天状态不错",
  "weather_key": "sunny"
}
```

说明：`user_id` 可省略，后端会按 token 自动绑定当前用户。

### 7.4 删除单天记录

- `DELETE /mood-calendar/{record_date}`

### 7.5 情绪打卡

- `GET /mood-calendar/checkins?limit=30`
- `POST /mood-calendar/checkins`

请求体示例：

```json
{
  "mood_score": 4,
  "stress_score": 2,
  "sleep_score": 3,
  "energy_score": 4,
  "note": "今天还可以"
}
```

### 7.6 趋势快照

- `GET /mood-calendar/trends?limit=90`
- `PUT /mood-calendar/trends/{snapshot_date}`

请求体示例：

```json
{
  "latest_risk_level": "moderate",
  "avg_mood_score": 3,
  "avg_stress_score": 2,
  "avg_sleep_score": 3,
  "phq9_latest_score": 14
}
```

## 8. Chat

### 8.1 创建聊天会话

- `POST /chat/sessions`
- 请求体：

```json
{
  "user_id": 1,
  "session_topic": "日常陪伴",
  "evaluation_session_id": null
}
```

### 8.2 查询聊天会话

- `GET /chat/sessions?user_id=1&limit=50`
- `GET /chat/sessions/{session_id}`

### 8.3 关闭会话

- `PATCH /chat/sessions/{session_id}/close`

### 8.4 发送消息

- `POST /chat/sessions/{session_id}/messages`
- 请求体：

```json
{
  "sender_type": "user",
  "content": "最近压力很大",
  "message_type": "text"
}
```

兼容写法：

```json
{
  "role": "assistant",
  "content": "你好",
  "message_type": "text"
}
```

其中 `assistant` 会自动映射为 `agent`。

### 8.5 获取消息

- `GET /chat/sessions/{session_id}/messages`

## 9. Health

- `GET /health`（全局，非 `/api/v1`）
- `GET /api/v1/health`
- `GET /api/v1/health/db`

建议前端启动时调用 `GET /api/v1/health/db` 作为联调连通性检查。

## 10. 前端接入建议

- 首次登录后保存 `access_token`，统一在请求拦截器中追加 `Authorization`。
- 情绪日历与报告页面不要再手动传其他用户 `user_id`。
- 问卷流程建议顺序：
  1. `POST /evaluations/sessions`
  2. `POST /evaluations/sessions/{session_id}/submit`
  3. `GET /reports/{report_id}/frontend`
